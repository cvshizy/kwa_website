KWA Website Handoff Guide

This guide explains how to hand off this project to another developer who will use Claude Code or a normal code editor to work on the site.

Project Overview

This is the KWA Art Center website.

Tech stack:

  • Next.js 14 App Router
  • React 18
  • TypeScript
  • Tailwind CSS
  • next-intl for Chinese/English routing
  • Sanity CMS for content
  • Vercel for global deployment
  • Alibaba Cloud ECS for China deployment

Main local URL:

http://localhost:3000/zh

Production domain:

https://kwartcenter.com

Recommended Handoff Method

Use GitHub collaboration instead of sending a copied project folder.

Recommended workflow:

  1. Invite the collaborator to the GitHub repository.
  2. Ask them to create a new branch for each task.
  3. Ask them to open a Pull Request.
  4. Review and merge the Pull Request yourself.
  5. Deploy after review.

Do not ask collaborators to commit directly to main unless they are fully trusted maintainers.

What The Collaborator Needs

The collaborator should install:

  • Node.js 18 or newer, Node 20 LTS recommended
  • npm
  • Git
  • Claude Code, if they will use Claude Code
  • A Claude.ai or Anthropic Console account
  • Access to GitHub
  • Optional: Sanity access, only if they need to edit CMS schema or content

Claude Code installation:

npm install -g @anthropic-ai/claude-code
claude doctor

Anthropic recommends not installing Claude Code with sudo npm install -g.

Clone And Run Locally

git clone git@github.com:cvshizy/kwa_website.git
cd kwa_website
npm ci
cp .env.local.example .env.local
npm run dev

Open:

http://localhost:3000/zh

Verify production build:

npm run build
npm run start

Environment Variables

Never send .env.local directly.

Create a local .env.local from .env.local.example:

cp .env.local.example .env.local

Typical local values:

NEXT_PUBLIC_SANITY_PROJECT_ID=qiafoam7
NEXT_PUBLIC_SANITY_DATASET=production
NEXT_PUBLIC_SANITY_API_VERSION=2024-01-01

SANITY_API_READ_TOKEN=

NEXT_PUBLIC_SITE_REGION=global
NEXT_PUBLIC_SITE_URL=http://localhost:3000
NEXT_PUBLIC_CONTACT_FORM_ACTION=https://formspree.io/f/xbdabrdw

SANITY_REVALIDATE_SECRET=local-dev-random-secret

GOOGLE_SITE_VERIFICATION=
BING_SITE_VERIFICATION=
BAIDU_SITE_VERIFICATION=

Notes:

  • SANITY_API_READ_TOKEN is optional for local frontend work if public reads are available.
  • Use a read-only Sanity token if a token is needed.
  • Do not share Sanity write tokens unless the collaborator must run migration/import scripts.
  • Do not share production deployment secrets by default.

Claude Code Usage

Start Claude Code inside the project:

cd kwa_website
claude

Recommended first prompt:

Read this Next.js project first. Do not modify files. Summarize the project structure, main pages, Sanity CMS integration, local run commands, and deployment flow.

Recommended safety prompt before edits:

Before changing files, give me a short plan. Do not edit .env.local, do not run deployment scripts, do not run scripts/import-* or scripts/migrate-*, and do not commit or push unless I explicitly approve.

Recommended task prompt example:

Change the Chinese homepage label from 特色展览 to 精选展览. Only modify the necessary file. Run npm run build after the change.

Development Workflow

For each task:

git checkout main
git pull origin main
git checkout -b feature/short-task-name
npm ci
npm run dev

After making changes:

npm run build
git status --short
git diff
git add <changed-files>
git commit -m "type: short description"
git push origin feature/short-task-name

Then open a Pull Request.

Commit message examples:

fix: update homepage copy
feat: add press pdf fallback
chore: update handoff documentation

Creating A Clean Zip Package

If GitHub access is not available, create a clean zip from the latest committed code:

git status --short

This zip includes committed project files only.

It does not include:

.env.local
node_modules

If uncommitted changes must be included, commit them to a branch first. Avoid hand-copying the whole folder unless absolutely necessary.

Sanity Access

Only grant Sanity access if the collaborator needs it.

Recommended permissions:

  • Viewer: can inspect content
  • Editor: can edit content
  • Developer: can edit schemas and Studio behavior
  • Owner: avoid unless they are a long-term maintainer

High-risk scripts that require SANITY_WRITE_TOKEN:

scripts/import-exhibitions.mjs
scripts/import-missing-pure-exhibitions.mjs
scripts/import-press.mjs
scripts/migrate-about-content.mjs
scripts/migrate-contact-content.mjs
scripts/migrate-home-content.mjs
scripts/migrate-press-content.mjs
scripts/patch-descriptions.mjs
scripts/patch-press-covers.mjs

Do not run these scripts on production CMS unless the change has been reviewed.

Vercel Deployment

Vercel deploys the global site from GitHub.

Most collaborators do not need Vercel access.

If access is needed:

  • Give Viewer for logs-only access.
  • Give Developer if they need to inspect environment variables or deployments.
  • Avoid Owner unless necessary.
  • Do not share Vercel tokens in chat.

China Deployment

The China site is deployed to Alibaba Cloud ECS with:

npm run deploy:cn

Do not run this without explicit permission.

The deploy script expects:

  • A clean local Git working tree
  • Local HEAD already pushed to origin/main
  • SSH access to the China server
  • Server-side env file at /var/www/shared/kwa_website/.env.local
  • Node, npm, pm2, and nginx already configured on the server

Recommended permission model:

  • Do not share root password.
  • Create a dedicated deploy user if a collaborator must deploy.
  • Use SSH keys instead of passwords.
  • Do not share Alibaba Cloud root account or AccessKey.

Review Checklist

Before accepting a collaborator's change:

npm run build
git diff main...HEAD

Check:

  • No .env.local or secrets are committed
  • No node_modules, .next, or build artifacts are committed
  • No migration/import script was run unintentionally
  • UI copy is correct in both Chinese and English where needed
  • Links and images still work
  • Sanity schema changes are intentional
  • Deployment scripts were not changed unexpectedly
